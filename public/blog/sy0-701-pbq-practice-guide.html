SY0-701 PBQ Practice Guide: Performance-Based Questions Explained

What PBQs are, how they are scored, and a step-by-step approach for each question type on exam day

June 2026 • 10 min read

By the Sleek Academia tutoring team

securityPart of our Complete CompTIA Security+ Prep Guide →

The Performance-Based Questions (PBQs) on the CompTIA Security+ SY0-701 exam are often the difference between a passing score and a retake. Multiple-choice questions test what you know; PBQs test whether you can actually do it — configure a firewall, analyse a log, set up a wireless network, or troubleshoot an access control issue in a simulated environment. They account for roughly 15–25% of your total score depending on the exam form, and they are almost always the section candidates describe as hardest.

This guide covers every PBQ format you will see on SY0-701, how each one is scored, and a repeatable approach for each type. If you are still early in your Security+ preparation, start with our overview of whether CompTIA Security+ is still worth it in 2026 and our full CompTIA Security+ prep guide for the complete domain breakdown.

What Exactly Are PBQs on the SY0-701?

PBQs are interactive questions that place you inside a simulated IT environment. Instead of picking the correct answer from four options, you perform a task: drag firewall rules into the correct order, click through a router configuration interface to apply ACLs, or identify vulnerabilities from a packet capture. CompTIA describes them as measuring application of knowledge rather than recall of facts.

The five SY0-701 domains all appear in PBQ format, but certain domains are heavier on simulations than others:

  • Domain 3 — Security Architecture: Implementing secure network designs, configuring firewalls and VPNs.
  • Domain 4 — Security Operations: Analysing logs, incident response scenarios, vulnerability scanning.
  • Domain 2 — Threats, Vulnerabilities, and Mitigations: Identifying attack types from traffic data or system indicators.
  • Domain 1 — General Security Concepts: Access control models and authentication configurations.
  • Domain 5 — Security Program Management: Occasionally appears as PBQs involving policy or compliance decisions.

The Four PBQ Formats on the SY0-701

1. Drag-and-Drop Build-Outs

You are given a set of items (firewall rules, security controls, configuration steps) and must arrange them in the correct order or into the correct categories. For example, you might be asked to place the phases of an incident response lifecycle — Preparation, Detection and Analysis, Containment, Eradication, Recovery, Post-Incident Activity — into the correct sequence.

How to approach: Look for the logical flow first. If it is a sequence, identify what starts the process and what ends it, then fill in the middle. If it is a classification task (sorting controls into preventive, detective, corrective), start with the items you are certain about and eliminate from there. The SY0-701 objective domains in the exam guide are a reliable reference for what belongs where.

2. Network Topology Configuration

This is the most intimidating PBQ type. You are shown a network topology diagram with empty slots or misconfigured devices and must drag correct devices, IP addresses, or security controls into place. A typical scenario: you see a small business network with a DMZ, internal LAN, and management network, and must place a firewall, web server, database server, and IDS sensor into the correct segments.

How to approach: Start with the external-facing components. The web server belongs in the DMZ. The database server belongs in the internal LAN. The management interface should have restricted access. Apply the principle of least privilege to every connection. If the scenario mentions compliance (PCI DSS, HIPAA), factor in which segments must be isolated. Almost all topology PBQs test whether you understand segmentation and the difference between what belongs publicly accessible versus what does not.

3. Log Analysis and Forensics

You are given a firewall log, system log, or packet capture output and asked to identify the type of attack, the affected system, or the misconfiguration. The SY0-701 exam frequently uses logs to test your ability to distinguish between a brute-force attack, a DDoS, SQL injection, and a man-in-the-middle attempt.

How to approach: Read the timestamp column first — do the events cluster in a short window? That suggests an automated attack. Look at source IPs: a single external IP with many failed authentication attempts is a brute force; many distributed IPs hitting the same port is a DDoS. Check the destination port: Port 443 with an unusual payload might indicate SSL/TLS abuse. Port 3306 (MySQL) outbound from the DMZ suggests SQL injection or data exfiltration. CompTIA expects you to recognise attack signatures from log output, not memorise every port number, but knowing the common service ports (22, 80, 443, 3389, 3306, 1433) helps significantly.

4. Command-Line and Remediation

You interact with a simulated command-line interface (CLI) or configuration panel to apply a fix. For example: you authenticate to a router, run show access-list, identify an overly permissive rule, and edit it. Or you connect to a Linux server and use iptables, chmod, or usermod commands to harden a system.

How to approach: Read the prompt carefully — it tells you exactly what the expected outcome is (e.g., “restrict SSH access to the 10.0.1.0/24 subnet”). Work backward: what command produces that result? The SY0-701 exam guide lists the command-line tools you need to know: ping, tracert, nslookup, netstat, ipconfig/ifconfig, nmap, iptables, and chmod. If you do not know the exact syntax, eliminate options that produce a different outcome. For example, chmod 777 gives full permissions to everyone — if the task says “restrict to owner only,” chmod 700 or chmod 755 is the correct choice.

PBQ Scoring: What You Need to Know

PBQs are not all-or-nothing. For drag-and-drop and topology questions, each correctly placed item earns partial credit. For log analysis, you may need to identify two indicators of compromise — identifying three out of four still earns most of the points. Multi-step CLI scenarios typically require completing every step to earn full credit, but the simulation gives you feedback (the interface changes when a step succeeds), so you know whether each action is working.

Because PBQs carry higher point values per question than multiple-choice, spending extra time on them is usually worth it. A good rule: allocate three to five minutes per PBQ and move on if stuck. The SY0-701 gives you 90 minutes for a maximum of 90 questions, and unfinished multiple-choice questions cost you points just as surely as unfinished PBQs do.

How to Practise PBQs Before Exam Day

Reading about PBQs will not prepare you for PBQs. You need hands-on practice in a safe environment that mimics the CompTIA simulation interface. Here is the most effective practice path:

  • Professor Messer's course notes and videos: Free resources that walk through every SY0-701 objective with examples. Watch the PBQ walkthroughs specifically.
  • Jason Dion's practice exams on Udemy: His PBQ simulations are widely regarded as the closest to the real exam interface. Take at least two full-length practice exams that include PBQs before scheduling your test.
  • CompTIA's official CertMaster Labs: These are expensive but they run in the actual PBQ environment. If you can afford one lab bundle, this is the best investment.
  • Home lab on VirtualBox: Set up a virtual network with pfSense, Kali Linux, and a Windows target. Practice configuring firewall rules in pfSense and analysing captures in Wireshark. The tools are free and the experience is transferable.

Exam-Day Strategy for PBQs

On exam day, the PBQs appear at the beginning of the test, before the multiple-choice questions. Many candidates spend too much time on them early and rush through the faster multiple-choice section when they could have earned more points there. A better strategy: flag difficult PBQs after 90 seconds and return to them after completing the rest of the exam. The Pearson VUE interface allows flagging and navigating between questions freely within a section.

Read each PBQ prompt twice before touching the simulation. The single biggest mistake is clicking into the interface without understanding what the question is asking. The prompt always states the objective clearly; the simulation is just the tool for demonstrating it.

Frequently Tested SY0-701 PBQ Scenarios

Based on exam feedback from recent test-takers (2025–2026), these are the most commonly appearing PBQ scenarios on the SY0-701:

  • Configure a firewall ACL: Allow HTTP/HTTPS to a web server, allow SSH from a management subnet, deny all else.
  • Set up a secure wireless network: Choose WPA3 or WPA2-AES, disable WPS, configure a captive portal for guests.
  • Analyse a phishing email header: Identify the spoofed sender, the actual originating IP, and the malicious URL.
  • Classify security controls: Sort given controls into preventive, detective, corrective, deterrent, and compensating categories.
  • Harden a Linux server: Disable root SSH login, remove unused user accounts, set file permissions, and enable logging.
  • Identify an attack from a packet capture: Distinguish between a SYN flood and a ping sweep based on TCP flags and packet timing.

PBQs are the hardest part of the SY0-701 — but they are also the most learnable. Every PBQ tests a specific skill listed in the exam objectives. If you can describe how to perform that skill step by step, the simulation is just executing what you already know. Practise the scenarios above, use the resources listed, and treat the PBQ section as an opportunity to demonstrate real competence rather than a hurdle to survive.

Frequently Asked Questions

How many PBQs are on the CompTIA Security+ SY0-701?

Between 3 and 6 Performance-Based Questions per exam form. They account for 15–25% of your total score. The exact number depends on the form you receive.

Are PBQs harder than the multiple-choice questions on the SY0-701?

Most candidates find PBQs harder because they test application rather than recall. However, each PBQ item (drag-and-drop slot, configuration step) earns partial credit, so even incomplete answers contribute to your score.

Can you skip PBQs and come back to them on the SY0-701?

Yes. The Pearson VUE interface lets you flag questions and navigate freely within each exam section. It is often strategic to flag a difficult PBQ and return after completing the multiple-choice section.
